Space Complexity

Significance Of Space Complexity

Space complexity, like time complexity is also an important parameter in determining the efficiency of a program. Here we will get to know about what is time complexity and its significance and how to calculate it.

What is Space Complexity?

Space Complexity of an algorithm or program is the total amount of memory or space taken by an algorithm as a function of the size of input. Let’s try in simple words, it is the memory required by an algorithm to execute a program and produce output. From this we can say space complexity is a parallel concept to time complexity.

Algorithm mainly uses memory space for three vital reasons:

1. Instruction Space

It is the quantity of memory required to save the compiled version of algorithm/program instructions.

2. Environmental Stack

Sometimes an algorithm (function) could also be called inside another algorithm (function). In this type of situation, current variables are pushed onto the system stack, where they wait for further execution and then the call to the inside algorithm (function) is made.

Let’s take an good example to understand more clearly, if a function X() calls function Y() inside it, then all the variables of the function X() will get stored on the system stack temporarily, while the function Y() is called and executed inside the function X().

3. Data Space

It defines the amount of space used by the variables and constants.

When we want to find the space complexity of an algorithm; we consider only the data space used in the algorithm i.e., we calculate only the space used by the variables and constants in the algorithm.

But many a time, Auxiliary Space is often get confused with Space Complexity. Auxiliary Space is the temporary space allocated by the algorithm while it is executed.

Space Complexity = Auxiliary Space + Input Space

Sometimes, while calculating the Space Complexity of any algorithm, we usually consider only Auxiliary space and we neglect the input space.



Notations Used in Space Complexity

We have already discussed Time Complexity, if you haven’t do surely read that… The time complexity of an algorithm is commonly expressed using the Big O notation as discussed, which is an asymptotic notation to represent the time complexity. Space complexity, is like time complexity, is often expressed asymptotically in Big-O notation, such as O(n), O(nlog(n)), O(n^2), etc., where n is the input size in units of bits needed to represent the input.
